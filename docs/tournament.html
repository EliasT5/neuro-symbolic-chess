Methodology

  • Every contestant starts at 1500 Elo.
  • K-factor is 40 for each model's first 30 rated games (or until rating reaches 2300), then 20. FIDE convention.
  • Self-play games are played for diagnostic purposes but do not move Elo.
  • Colors are alternated by a round-robin scheduler so no model accumulates a White advantage.
  • Time control: 60s per move soft cap; per-move token budget guards against runaway cost.
  • Every move is validated by the chess_core.guardrails primitives — illegal proposals are retried once, then the engine falls back to a uniformly random legal move (counted as a fallback).

Thinking variants

Each base model is also entered as a (thinking) variant. The thinking variant prepends a uniform prompt scaffold asking the model to reason inside <thinking>...</thinking> tags before committing to a move. The block is captured (for later inspection) and stripped before the move is parsed.

This is provider-agnostic by design: every model gets the same scaffold so the Elo gap between base and thinking is a clean measurement of "what a few hundred extra reasoning tokens buy in chess." Models with native extended thinking (Opus, Sonnet 4.6, Model-G 3 Pro/Auto, Model-O 5.5/5.3 Model-X) will simply benefit more from it.

What we measure

Elo & score %
Strength as inferred from match results.
Illegal-move rate
Fraction of LLM proposals the guardrails rejected. Drunken-Robot Syndrome, but for legality.
Fallback rate
Fraction of moves that ended up at the random-legal escape hatch (after one retry failed).
$/Elo
Total dollars spent divided by Elo points gained over 1500. The value chart — the cheap-but-decent winner.
Centipawn loss
Average loss-vs-best-move per move, computed post-hoc with Stockfish. The gold-standard chess strength metric.
Style fingerprint
Capture %, check %, average game length, decisiveness — what the model likes to play, regardless of strength.

Why we want both extremes

The strongest model tells us which base has the deepest pretrained chess prior — it's the cheapest path to a competent System 1. The weakest model tells us which base is the cleanest slate, the best candidate for a from-scratch fine-tune that won't be fighting prior knowledge. Either end of the distribution is research-useful; the middle is the part we can ignore.
